Topic 10: Error Handling

10.1 What is Error Handling?

Error handling allows you to manage and respond to errors in your code gracefully without crashing the program.

10.2 Types of Errors

  • Syntax Errors: Mistakes in the code structure.
  • Runtime Errors: Errors that occur while the program is running (e.g., division by zero).
  • Logical Errors: The program runs but produces incorrect results.

10.3 Try and Except

The try block lets you test a block of code for errors. The except block lets you handle the error.

try:
  result = 10 / 0
except ZeroDivisionError:
  print("Cannot divide by zero!")

10.4 Catching Multiple Exceptions

try:
  num = int("abc")
except ValueError:
  print("Invalid number!")
except Exception as e:
  print(f"An error occurred: {e}")

10.5 Else and Finally

  • else: Executes if no errors occur.
  • finally: Executes no matter what, useful for cleanup actions.
try:
  print("Trying something...")
except:
  print("An error occurred!")
else:
  print("No errors!")
finally:
  print("This always runs!")

10.6 Raising Custom Errors

You can raise your own exceptions using raise keyword.

age = -5
if age < 0:
  raise ValueError("Age cannot be negative!")

10.7 Summary

  • try and except help handle errors without stopping the program.
  • else runs if no errors occur.
  • finally always executes for cleanups.
  • Custom errors can be raised using raise.
